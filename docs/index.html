01System Architecture

We don't maintain a skill catalog or detection logic. autoskills does all the heavy lifting — we only provide the OpenCode integration layer.

⚡

Powered by autoskills

All technology detection, skill mapping, registry management, and installation comes from midudev/autoskills. SHA-256 verified, curated registry.

🪡

Thin Wrapper

Under 300 lines of TypeScript. A custom OpenCode tool + setup CLI. No duplicate logic. No maintenance burden.

🪙

Zero Token Cost

Detection, selection, and installation run outside the agent. Only the initial invocation costs tokens (~200). The rest is direct terminal interaction.

02Features

AI-Powered Execution

Your OpenCode agent invokes the autoskills tool. The heavy lifting runs natively, not through the LLM.

Near-Zero Latency

No extra network requests. Skills are downloaded and verified by autoskills' own pipeline.

Interactive Selection

Use autoskills' TUI to pick exactly which skills you want — not all-or-nothing like headless installers.

Global or Per-Project

Setup once globally with --global, or keep it scoped with --local.

Security First

No postinstall scripts. No runtime dependencies beyond @opencode-ai/plugin (peer). Path traversal prevention in setup.

Multi-Platform

Windows, macOS, and Linux. CI-tested on all three with GitHub Actions.

03Installation & Usage

One-time setup. Then /autoskills in any project.

Terminal — Setup (global, one time)
$ npx opencode-autoskills setup
Terminal — Per-project setup
$ npx opencode-autoskills setup --local
OpenCode TUI — After setup
/autoskills

# Or just tell the agent:
> install skills for this project
What happens next
→ OpenCode AI reads the command
→ Invokes the autoskills tool
→ Tool responds: "run npx autoskills"
→ AI uses bash → autoskills TUI opens
→ You pick skills interactively
→ Installed to .agents/skills/
→ OpenCode discovers them automatically

04Configuration

Environment variables and requirements.

Variable / SettingDescription
Node.js Required >= 22.6.0 (enforced by autoskills)
@opencode-ai/plugin Peer dependency >= 1.0.0 (provided by OpenCode)
--global Setup flag (default). Installs to ~/.config/opencode/commands/
--local Setup flag. Installs to .opencode/commands/ in current project
opencode.json Setup adds opencode-autoskills to the OpenCode plugin array so the custom tool is available
.agents/skills/ Skills install path (autoskills default, OpenCode compatible)
GITHUB_TOKEN Optional. Passed through to autoskills for increased API rate limits
